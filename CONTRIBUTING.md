# Contributing

Thanks for the interest in improving this project.

## Ground Rules

- External code contributions are considered only if the contributor agrees to
  the repository's contributor license agreement in [`CLA.md`](./CLA.md).
- Submitting a pull request does not grant you any commercial rights in this
  repository.
- Do not add copied third-party PDFs, issue exports, documentation snapshots,
  or similar archive material to the public repository unless redistribution is
  clearly permitted and the material is documented in
  [`THIRD_PARTY_NOTICES.md`](./THIRD_PARTY_NOTICES.md).
- Do not relabel vendored or third-party code as solely original project code.

## Before Opening A Pull Request

- read the repository [`LICENSE`](./LICENSE)
- read the commercial-use summary in
  [`COMMERCIAL-LICENSING.md`](./COMMERCIAL-LICENSING.md)
- confirm you have the right to submit the code
- confirm you are willing to agree to the CLA if the maintainer requests it for
  merge

## Review Expectations

The maintainer may require extra review for:

- legal or licensing changes
- third-party notice changes
- vendored package changes
- security-sensitive changes

## Security Reports

For vulnerability reporting, follow the process in
[`.github/SECURITY.md`](./.github/SECURITY.md).
